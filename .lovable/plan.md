## План: страница «Кейсы»

### 1. Данные кейсов
Создать `src/lib/cases-data.ts` с массивом из 12 кейсов. Структура:
```ts
type Case = {
  slug: string;
  title: string;       // Название кейса
  niche: string;       // Ниша
  problem: string;     // Проблема
  solution: string;    // Решение ABP
  results: string[];   // Результаты (массив пунктов)
};
```
Контент — 12 кейсов из задания (Эликсир роста, Умная мобильность, Агро-экспансия, Улыбка без страха, Алхимия чистоты, Билет в Лигу Плюща, Мясная философия, Сила источника, Продавая будущее, Инженерная точность, Каменная уверенность, Цифровой шоу-рум).

### 2. Маршрут `/cases`
Создать `src/routes/cases.tsx` (TanStack file-based route):
- `head()` с уникальными meta: title «Кейсы — ABP DIGITAL», description, og:title, og:description.
- Хедер страницы: H1 «Кейсы» + короткий подзаголовок.
- Сетка карточек: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-6` (4 в ряд на десктопе, как просил пользователь; адаптивно сворачивается на планшете/мобайле).
- Карточка: бордер `border-hairline`, padding, скругление. Содержит:
  - Ниша — мелким моно-капсом сверху (`text-muted-foreground`).
  - Название кейса — `font-display` крупно.
  - Кнопка «Посмотреть» снизу, открывает модалку.
- Состояние выбранного кейса — `useState<Case | null>`. Один общий `<Dialog>` на странице, контент которого подставляется из выбранного кейса.

### 3. Модальное окно
Используем существующий `@/components/ui/dialog`. Внутри:
- `DialogTitle` — название кейса.
- `DialogDescription` — ниша.
- Секции «Проблема», «Решение ABP», «Результаты» (последняя — `<ul>` со списком).
- Прокручиваемый контейнер `max-h-[80vh] overflow-y-auto` на случай длинного текста.

### 4. Навигация
- `src/components/site-header.tsx`: добавить `<Link to="/cases">Кейсы</Link>` в десктопное меню (между «Услуги» и «Команда») и в мобильное меню (Sheet).
- `src/components/site-footer.tsx`: добавить пункт «Кейсы» в блок «Навигация».

### 5. Проверка
После сборки — открыть `/cases`, проверить что 4 карточки в ряд на десктопе, модалка открывается с правильным контентом, ссылки в шапке/футере работают.
